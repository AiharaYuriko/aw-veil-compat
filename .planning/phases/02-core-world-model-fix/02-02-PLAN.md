<objective>
Replace the failed Veil ResourceProvider wrapping approach with Pattern 1: dedicated GL shader program for AW VBO rendering. 

Purpose: AW's VBO rendering fails under Veil because ShaderUniforms.end() reads GL_CURRENT_PROGRAM, and the current program (Veil's) does not have AW's 7 uniform declarations — glGetUniformLocation returns -1, uniforms are silently dropped, models don't render. Pattern 1 solves this architecturally: give AW its own shader program that unequivocally declares all 7 AW uniforms. Inject save/restore at ShaderUniforms.end() to bind AW's program before uniform application and restore the original program after.

This approach is Veil-agnostic (works with ANY mod that changes the active shader), adds only 2 glUseProgram calls per render cycle (negligible overhead per PERF-01), and eliminates the root cause at the architectural level.

Output:
- AwDedicatedProgram.java — GLSL shader program factory (vertex+fragment shader creation, compile/link with error handling, matrix upload from RenderSystem)
- AwDedicatedShaderMixin.java — @Inject at ShaderUniforms.end() HEAD (save+bind+upload) and RETURN (restore) with re-entrancy counter
- Cleaned mixin configs — AW config registers AwDedicatedShaderMixin, Veil config no longer registers VeilShaderResourceMixin
- Removed failed-approach files: VeilShaderResourceMixin.java, AwShaderUniformInjector.java (dead code after approach change)
</objective>

    ---- GLSL SOURCES ----

    VERTEX_SHADER constant:
    ```glsl
    #version 150 core

    layout(location = 0) in vec3  Position;
    layout(location = 1) in vec4  Color;
    layout(location = 2) in vec2  UV0;
    layout(location = 3) in ivec2 UV1;
    layout(location = 4) in ivec2 UV2;
    layout(location = 5) in vec3  Normal;

    uniform mat4 ModelViewMat;
    uniform mat4 ProjMat;
    uniform vec4 ColorModulator;

    uniform mat4 aw_ModelViewMatrix;
    uniform mat4 aw_TextureMatrix;
    uniform mat4 aw_OverlayTextureMatrix;
    uniform mat4 aw_LightmapTextureMatrix;
    uniform mat3 aw_NormalMatrix;
    uniform vec4 aw_ColorModulator;
    uniform int  aw_MatrixFlags;

    out vec4 vertexColor;
    out vec2 texCoord0;
    out vec2 texCoord1;
    out vec2 texCoord2;
    out float vertexDistance;

    void main() {
        vec4 awPos = aw_ModelViewMatrix * vec4(Position, 1.0);
        vec4 viewPos = ModelViewMat * awPos;
        gl_Position = ProjMat * viewPos;
        vertexDistance = length(viewPos.xyz);
        vertexColor = Color * ColorModulator;
        texCoord0 = UV0;
        texCoord1 = UV1;
        texCoord2 = UV2;
    }
    ```

    FRAGMENT_SHADER constant:
    ```glsl
    #version 150 core

    in vec4 vertexColor;
    in vec2 texCoord0;
    in vec2 texCoord1;
    in vec2 texCoord2;
    in float vertexDistance;

    uniform sampler2D Sampler0;

    out vec4 fragColor;

    void main() {
        vec4 color = texture(Sampler0, texCoord0) * vertexColor;
        if (color.a < 0.1) discard;
        fragColor = color;
    }
    ```

    ---- PROGRAM CREATION ----

    Method `public static int create()`:
    1. Compile vertex shader: `GL20.glCreateShader(GL20.GL_VERTEX_SHADER)`, `GL20.glShaderSource()`, `GL20.glCompileShader()`. Check compile status via `GL20.glGetShaderi(shader, GL20.GL_COMPILE_STATUS)`. On failure: get info log via `GL20.glGetShaderInfoLog()`, delete shader, throw RuntimeException with the log.
    2. Compile fragment shader: same process with `GL20.GL_FRAGMENT_SHADER`.
    3. Create program: `GL20.glCreateProgram()`. Attach both shaders via `GL20.glAttachShader()`. Link via `GL20.glLinkProgram()`. Check link status via `GL20.glGetProgrami(program, GL20.GL_LINK_STATUS)`. On failure: get info log, delete program and shaders, throw RuntimeException.
    4. Detach and delete shader objects: `GL20.glDetachShader()` then `GL20.glDeleteShader()` for both. (Shaders remain usable for linking until detached, but after linking they are no longer needed.)
    5. Return program ID.

    Helper `private static int compileShader(int type, String source)`:
    - Create shader, set source, compile.
    - If `GL_COMPILE_STATUS == GL_FALSE`: get info log, delete shader, throw RuntimeException("Shader compile failed: " + log).
    - Return shader ID.

    ---- MATRIX UPLOAD ----

    Method `public static void uploadMatrices(int program)`:
    - Capture current matrices from RenderSystem:
      `Matrix4f projMat = RenderSystem.getProjectionMatrix();`
      `Matrix4f modelViewMat = RenderSystem.getModelViewMatrix();`
      `Vector4f shaderColor = RenderSystem.getShaderColor();`
    - Use LWJGL MemoryStack for zero-allocation float buffer:
      ```java
      try (MemoryStack stack = MemoryStack.stackPush()) {
          FloatBuffer buf = stack.mallocFloat(16);

          projMat.get(0, buf);
          buf.flip();
          int loc = GL20.glGetUniformLocation(program, "ProjMat");
          if (loc >= 0) GL20.glUniformMatrix4fv(loc, false, buf);

          modelViewMat.get(0, buf);
          buf.flip();
          loc = GL20.glGetUniformLocation(program, "ModelViewMat");
          if (loc >= 0) GL20.glUniformMatrix4fv(loc, false, buf);

          loc = GL20.glGetUniformLocation(program, "ColorModulator");
          if (loc >= 0) {
              GL20.glUniform4f(loc, shaderColor.x(), shaderColor.y(), shaderColor.z(), shaderColor.w());
          }
      }
      ```
    - Note: `Matrix4f.get(int index, FloatBuffer buffer)` stores the 4x4 matrix in column-major order starting at the buffer's position + index. With index=0, it stores at the current position. After storing, buffer position advances by 16. `buf.flip()` resets position to 0 for glUniformMatrix4fv.

    ---- HEAD INJECTION (onEnd) ----

    Annotation: `@Inject(method = "end", at = @At("HEAD"), require = 0)`
    Callback: `private static void onEnd(CallbackInfo ci)`

    Logic (in order):

    1. Guard: `if (!ModDetector.isAWLoaded()) return;`

    2. Re-entrancy guard: `if (reentrantDepth > 0) { reentrantDepth++; return; }`
       - If end() is called recursively, increment depth and skip the save/bind.
       - The TAIL will decrement depth on each exit, so the outermost call handles restore.

    3. Lazy init (awProgram == 0):
       ```java
       if (awProgram == 0) {
           if (!RenderSystem.isOnRenderThread()) return; // GL context not available yet
           try {
               awProgram = AwDedicatedProgram.create();
           } catch (Exception e) {
               awProgram = -1; // permanent failure
               return;
           }
       }
       ```
       - On success: awProgram = valid program ID (reused for session).
       - On failure: awProgram = -1 (all future calls skip immediately).

    4. Skip check: `if (awProgram <= 0) return;` — program creation failed, do nothing.

    5. Save current program:
       ```java
       int current = GL20.glGetInteger(GL20.GL_CURRENT_PROGRAM);
       if (current == awProgram) return; // already our program bound, nothing to do
       savedProgram = current;
       ```

    Callback: `private static void onEndReturn(CallbackInfo ci)`

    ---- CRITICAL SAFETY PATTERNS ----

    1. **Re-entrancy race:** savedProgram is set BEFORE glUseProgram (line order matters). If GL20.glUseProgram fails (throws exception), savedProgram still points to the original program. The TAIL fires in the finally block (Mixin @At("RETURN") semantics) and restores correctly.

    2. **Poisoned state recovery:** If an exception propagates through end() and TAIL still fires (it does per Mixin semantics), both savedProgram and reentrantDepth are reset. The fix state is clean for the next call.

    3. **Thread safety:** lazy init guards with RenderSystem.isOnRenderThread(). All shader operations happen on the render thread. Static fields are only read/written from the render thread at ShaderUniforms.end() time.

    4. **Coexistence with probe:** ShaderUniformsProbeMixin also @Inject at end() HEAD. Both fire. The probe logs what GL_CURRENT_PROGRAM is when its handler runs:

       - If our mixin fires before probe: logs AW's dedicated program
       Either is fine. Our mixin does not need explicit priority ordering because the final GL_CURRENT_PROGRAM when AW's method body runs is ours (set by whichever mixin fires last).
